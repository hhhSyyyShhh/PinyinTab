# PinyinTab 第二版功能说明

## 第二版解决的问题

第一版能够在当前目录中把拼音补全为真实中文名称，但对文件和目录没有区分，而且只稳定处理单层路径。

例如真实存在目录 `你好/` 时，第一版可能把：

```bash
cat nihao<Tab>
```

错误地补全为：

```bash
cat 你好/
```

目录末尾的 `/` 本身不是错误，它表示该候选是目录，方便继续输入下一层。错误在于 `cat` 需要普通文件，插件不应该把目录提供给它。

第二版增加了命令上下文和多级路径解析。

## 命令类型过滤

### 只补目录

`cd`、`rmdir` 只返回目录：

```bash
cd ceshimulu<Tab>
# cd 测试目录/
```

目录后保留 `/`，并且不自动添加空格，这样可以继续输入下一层。

### 只补文件

`cat`、`python3`、`julia`、`node`、`javac`、`rustc`、`clang`、`swiftc` 等只返回普通文件：

```bash
cat ceshimulu<Tab>
```

如果 `测试目录/` 是唯一匹配项，插件也不会把它补给 `cat`，原命令行保持不变。

### 文件和目录都可以

`cp`、`mv`、`rm`、`vim` 等保留文件和目录候选，因为这些命令可能处理两者。

## 多级嵌套路径

第二版会逐层解析路径中的拼音目录组件。用户不必先逐层完成补全，可以直接输入：

```bash
python3 ceshimulu/neibujiao<Tab>
```

插件先把 `ceshimulu` 唯一解析为真实目录 `测试目录`，再进入该目录匹配 `neibujiao`，最终得到：

```bash
python3 测试目录/内部脚本.py
```

中文父目录和拼音子名称也可以混合：

```bash
python3 测试目录/neibujiao<Tab>
```

## 空输入列出下一层

目录末尾什么都不输入，直接按 `Tab`，会列出下一层符合当前命令类型的候选：

```bash
cd ceshimulu/<Tab>
cat ceshimulu/neibuwenjianjia/<Tab>
```

可能分别补全为：

```bash
cd 测试目录/内部文件夹/
cat 测试目录/内部文件夹/深层说明.txt
```

Backspace（退格键）仍然只负责删除字符。删除到目录斜杠后，再按 `Tab` 才会列出当前层内容。把退格键改成补全键会破坏终端的常规操作，因此第二版不修改 Backspace 的标准行为。

## 重名与歧义

解析中间目录时，真实中文名优先，其次是完整拼音和首字母。只有唯一匹配时才自动进入下一层；如果多个目录的拼音相同，插件不会随机选择，用户需要先选择正确目录。

## 文件主体缩写与扩展名

2.0.1 修复了“缩写后直接写扩展名”无法匹配的问题。插件现在分开比较文件主体和扩展名：

```text
jiujiu.py  → 九九乘法表.py
cfkjb.py   → 乘法口诀表.py
cfkjb.jl   → 乘法口诀表.jl
```

同时增加了 `python3.10` 到 `python3.14` 等带版本号 Python 命令的补全注册。

当前插件仍然在按下 `Tab` 时转换路径。直接输入拼音后按回车，Shell 会把拼音当作真实文件名，因此会报告文件不存在。

## Java 类名适配

`javac` 接收源码文件路径，但 `java` 接收类名，不能带 `.class`：

```bash
javac cfkjb.java<Tab>  # javac 乘法口诀表.java
java cfkjb<Tab>        # java 乘法口诀表
```

2.0.2 为 `java` 增加了专用模式：只匹配当前目录的 `.class` 文件，补全时去掉 `.class`，并忽略包含 `$` 的内部类文件。带 `package` 的 Java 类需要使用完整包名，暂不属于普通路径补全范围。

## 第二版测试重点

- `cat` 不再补全目录。
- `cd` 不再补全普通文件。
- 空结果不会在 Zsh 中变成一个虚假的 `/`。
- 支持全拼父目录加拼音子文件。
- 支持中文父目录加拼音子文件。
- 支持目录末尾空输入后按 `Tab`。
- 支持三层及以上的混合路径。
- 支持带空格的中文文件名。
- Bash 和 macOS Zsh 都有自动化回归测试。
